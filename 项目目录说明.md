## 一、项目是干什么的

**Agnes AI Studio** 是一个基于 ModelScope（魔搭）API 的本地 AI 工作台，主要功能包括：

- **AI 对话**：支持多个大语言模型（Qwen、DeepSeek、GLM、MiniMax、Kimi、Step 等）。
- **AI 生图**：支持多个文生图模型（Z-Image、Qwen-Image、Kolors、Hunyuan、FLUX、SDXL、HiDream 等）。
- **图像与视频**：主页面 `Agnes.html` 是一个单文件应用，包含图像与视频相关功能。

由于浏览器无法直接跨域请求 ModelScope API，项目通过**本地代理服务器**解决 CORS 问题，并代理 API 请求。

项目同时提供了一个 **Cloudflare Worker** 版本，用于在线部署（从 GitHub raw 代理资源）。

### 技术栈

- **前端**：原生 HTML / CSS / JavaScript（无框架），深色主题，单文件应用为主。
- **本地代理**：
  - 主版本：Node.js（`server.js`，端口 8000）。
  - 早期版本：Python（`ai-studio/proxy_server.py`，端口 8765）。
- **在线部署**：Cloudflare Worker（`cf-worker/`）。
- **运行环境**：Windows（启动脚本为 `.bat`）。

## 三、各文件夹与文件详解

### 根目录文件

| 文件 | 用途 |
|------|------|
| `Agnes.html` | **主应用页面**。单文件应用（约 327KB），包含完整的 HTML/CSS/JS，深色主题，含图像与视频功能。是当前主要维护的版本。 |
| `index.html` | 入口跳转页。仅做一件事：把 `/` 跳转到 `/Agnes.html`。 |
| `server.js` | **Node.js 本地代理服务器**（端口 8000）。功能：① 静态文件服务；② ModelScope API 代理（`/api/modelscope/v1/...`）；③ 通用代理（`/api/proxy?target=URL`）；④ 下载代理（`/dl?url=URL`）；⑤ 启动时自动清除 `Agnes.html` 的 BOM。 |
| `启动.bat` | Windows 一键启动脚本。自动查找 Node.js，后台启动 `server.js`，并打开浏览器访问 `http://localhost:8000/Agnes.html`。 |
| `.gitignore` | 忽略 `副本/`、`gitee-2.0/`、`node_modules/`、`.DS_Store`。 |
| `项目规则与错误记录.md` | **开发规则与错误记录**。每位 AI 必读，详见文档内说明。 |
| `项目目录说明.md` | 本文档。 |

### `ai-studio/` —— 早期版本

这是项目的早期版本，使用 **Python 代理服务器** + **分离的前端文件**。当前主版本（根目录的 `Agnes.html` + `server.js`）已取代它，但此目录保留作为参考和 Skill 文档载体。

| 文件 | 用途 |
|------|------|
| `SKILL.md` | **ModelScope AIGC 前端集成开发 Skill 文档**。记录了 14 条踩坑经验和完整的实现方案，是本项目最重要的知识沉淀之一。修改本目录代码前建议先阅读。 |
| `index.html` | 早期前端页面（对话 + 生图）。 |
| `app.js` | 早期前端逻辑。 |
| `styles.css` | 早期深色主题样式。 |
| `proxy_server.py` | Python 代理服务器（端口 8765），解决 ModelScope API 的 CORS 问题。 |
| `start.bat` | 早期启动脚本（Python 版）。 |

> **注意**：新功能开发应在根目录的 `Agnes.html` 上进行，不要继续在 `ai-studio/` 上开发。`ai-studio/` 主要保留 `SKILL.md` 作为知识文档。

### `cf-worker/` —— Cloudflare Worker 在线部署版本

用于将项目部署到 Cloudflare Workers，实现在线访问。Worker 会从 GitHub raw 仓库代理资源，并把 `/` 重定向到 `/Agnes.html`。

| 文件 | 用途 |
|------|------|
| `src/index.js` | Worker 主代码。逻辑：`/` 和 `/index.html` 重定向到 `/Agnes.html`；其他路径代理到 `https://raw.githubusercontent.com/xinhaocai5-hue/agnes-ai/main` + 路径。设置 CORS 头，并修正 HTML 文件的 Content-Type。 |
| `wrangler.toml` | Wrangler 部署配置。Worker 名称为 `agnes-ai`，入口 `src/index.js`。 |
| `package.json` | 依赖配置，仅含 `wrangler` 作为 devDependency，脚本 `npm run deploy` 用于部署。 |
| `package-lock.json` | 依赖锁定文件。 |
| `.dev.vars` | 本地开发环境变量（可能含敏感信息，不要提交）。 |

> **部署方式**：在 `cf-worker/` 目录下执行 `npm run deploy`（即 `wrangler deploy`）。

### `副本/` —— 文件备份存放处

用于存放**文件修改前**的备份。每次修改已存在的文件前，必须先复制一份到此文件夹，命名规则见 [项目规则与错误记录.md](./项目规则与错误记录.md) 的"开发规则第 1 条"。

- 此文件夹已在 `.gitignore` 中忽略，不会提交到仓库。
- 新建文件不需要备份。
- 此文件夹由 AI 在开发过程中自动维护，不需要用户手动管理。

## 四、典型工作流程

### 启动项目（本地开发）

1. 双击 `启动.bat`，或手动执行 `node server.js`。
2. 浏览器访问 `http://localhost:8000/Agnes.html`。
3. 在左侧设置面板填入 ModelScope API Key，选择模型，开始使用。

### 修改主应用 `Agnes.html`

1. **先备份**：复制 `Agnes.html` 到 `副本/`，按规则重命名（如 `Agnes.html_v1_初始化备份.html`）。
2. **再修改**：编辑 `Agnes.html`。
3. **测试**：重启 `server.js`，刷新浏览器验证。
4. **记录**：如遇新坑，追加到 [项目规则与错误记录.md](./项目规则与错误记录.md)。

### 部署到 Cloudflare Worker

1. 进入 `cf-worker/` 目录。
2. 执行 `npm install`（首次）。
3. 执行 `npm run deploy`。
4. 确保 GitHub 仓库 `xinhaocai5-hue/agnes-ai` 的 `main` 分支已推送最新代码（Worker 从此代理资源）。
