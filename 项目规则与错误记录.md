# 项目规则与错误记录

> 本文档用于存放本项目的开发规则，以及 AI 在开发过程中曾犯过的错误，供后续每一位接手的 AI 阅读和遵守。
>
> **每位 AI 在开始任何工作前，必须完整阅读本文档。**
> **遇到新的规则或踩坑，应及时追加到本文档对应章节。**

---

## 一、开发规则（必须遵守）

### 1. 修改文件前必须先备份

每次要修改一个已存在的文件时，**必须先复制一份原文件**到 `副本/` 文件夹，并按以下规则重命名：

```
副本/<原文件名>_v<版本号>_<修改内容简述>.<原扩展名>
```

- **版本号**：从 `1` 开始递增；若 `副本/` 中已存在该文件的历史版本，取最大版本号 +1。
- **修改内容简述**：用简短的中文或英文描述本次修改的内容，单词之间用 `-` 连接，不要包含空格和特殊符号。
- **示例**：
  - `副本/Agnes.html_v3_修复生图按钮无法点击.html`
  - `副本/server.js_v2_增加下载代理路由.js`
  - `副本/app.js_v1_初始化备份.js`

注意事项：
- 新建文件不需要备份（因为没有原文件）。
- 备份的是**修改前**的版本，不是修改后的版本。
- 备份后才能对原文件进行修改。
- `副本/` 文件夹已在 `.gitignore` 中忽略，不会提交到仓库。

### 2. 界面设计规则

- **不要使用蓝紫色**作为主色调或强调色。
  - 当前项目主色调为黑灰系（`--bg-primary: #0a0a0a`，`--accent: #eaeaea`），保持这一风格。
- **不要使用 emoji 表情**。
  - 界面图标一律使用文字、SVG、Unicode 几何符号（如 `≡`、`✕`、`➤`）或图标字体，不要用 emoji。
  - 文档和代码注释中也尽量避免 emoji。

### 3. 文件编码与 BOM

- 所有文本文件统一使用 **UTF-8 无 BOM** 编码保存。
- `server.js` 启动时会自动清除 `Agnes.html` 的 BOM，但不要依赖这个机制——保存时就应当避免产生 BOM。

### 4. Windows 启动脚本（.bat）

- bat 文件中**不要使用中文**（包括 echo 和注释），避免 GBK/UTF-8 编码导致的乱码。
- bat 开头必须加 `cd /d "%~dp0"` 切换到脚本所在目录。
- 使用 `start /b` 在后台运行服务，避免弹出多个黑窗口。
- 打开浏览器用 `start http://...` 或 `explorer "http://..."`。

### 5. 代理服务器与 CORS

- 浏览器无法直接跨域请求 ModelScope API，**必须通过本地代理**。
- 代理服务器在复制上游响应头时，**必须跳过所有 CORS 相关头**（`access-control-allow-origin` 等），避免重复。
- 代理必须实现 `OPTIONS` 预检请求处理，并允许所有自定义请求头。

### 6. JavaScript 初始化顺序

- `debugConsole.init()` 必须在 `loadSettings()` 之前调用，否则按钮会无法点击（详见错误记录 #1）。
- 所有输入框必须监听 `input` / `change` 事件，实时同步到 `state` 对象。

### 7. API 端点区分

- 对话：`POST /v1/chat/completions`
- 生图：`POST /v1/images/generations`
- 任务查询：`GET /v1/tasks/{task_id}`

生图相关请求必须携带：
- 提交任务时：`X-ModelScope-Async-Mode: true`
- 轮询任务时：`X-ModelScope-Task-Type: image_generation`

## 二、错误记录（曾踩过的坑）

> 以下记录 AI 在本项目中实际犯过的错误。每条包含：现象、原因、解决方法、如何避免。
> 新增记录时请按编号递增，并标注日期。

### #1 JavaScript 初始化顺序错误导致按钮无法点击

- **日期**：早期开发
- **现象**：页面加载后所有按钮都点不了，控制台无任何报错。
- **原因**：`loadSettings()` 在 `debugConsole.init()` 之前被调用，但 `loadSettings()` 内部使用了 `debugConsole.log()`，此时 `debugConsole` 的 DOM 元素还未初始化（为 null），导致 JS 运行时错误，后续的 `bindEvents()` 没有执行。
- **解决**：调整初始化顺序为 `debugConsole.init()` → `loadSettings()` → `bindEvents()` → `updateUI()` → `checkProxyStatus()`。
- **如何避免**：任何依赖 DOM 的函数调用前，必须确认其依赖的 DOM 已初始化；初始化函数之间有依赖时，顺序不能随意调整。

### #2 CORS 响应头重复导致浏览器拒绝响应

- **日期**：早期开发
- **现象**：浏览器报 `The 'Access-Control-Allow-Origin' header contains multiple values '*, *', but only one is allowed`。
- **原因**：代理服务器自己添加了 `Access-Control-Allow-Origin: *`，同时上游 ModelScope 响应也返回了同样的头，导致响应中有两个重复的 CORS 头。
- **解决**：代理在复制上游响应头时，跳过所有 CORS 相关头。
- **如何避免**：写代理时，永远要明确区分"自己添加的头"和"从上游复制的头"，二者不能重复。

### #3 CORS 预检未处理自定义请求头

- **日期**：早期开发
- **现象**：`Request header field x-modelscope-async-mode is not allowed by Access-Control-Allow-Headers in preflight response.`
- **原因**：代理未实现 `OPTIONS` 方法，或 `Access-Control-Allow-Headers` 未允许自定义头。
- **解决**：代理实现 `do_OPTIONS` / `OPTIONS` 处理，`Access-Control-Allow-Headers` 设为 `*`。
- **如何避免**：只要前端使用了非标准请求头，就必须确保代理正确响应预检请求。

### #4 API Key 输入后未实时同步到 state

- **日期**：早期开发
- **现象**：输入了 API Key，但发送消息时提示"请先设置 API Key"。
- **原因**：API Key 只在点击"保存设置"按钮时才同步到 `state` 对象，但发送消息时检查的是 `state` 中的值。
- **解决**：输入框监听 `input` 事件，实时同步到 `state`。
- **如何避免**：UI 输入与状态对象之间必须建立实时同步机制，不能依赖"保存"按钮。

### #5 生图 API 端点用错

- **日期**：早期开发
- **现象**：生图请求返回 404 或返回对话结果。
- **原因**：使用了对话 API 端点 `/v1/chat/completions` 来生图。
- **解决**：生图必须使用 `/v1/images/generations`。
- **如何避免**：调用 API 前先确认端点与功能匹配，参考本文档"开发规则"第 7 条。

### #6 异步任务轮询缺少正确的请求头

- **日期**：早期开发
- **现象**：提交生图任务成功获得 task_id，但轮询任务状态时返回 404 或异常。
- **原因**：轮询时缺少 `X-ModelScope-Task-Type` 头。
- **解决**：轮询请求必须携带 `X-ModelScope-Task-Type: image_generation`。
- **如何避免**：异步任务相关的请求头容易遗漏，提交和轮询的请求头不同，需分别确认。

### #7 代理服务器连接中断崩溃

- **日期**：早期开发
- **现象**：浏览器取消请求或页面刷新时，代理服务器抛出 `ConnectionAbortedError` 并可能崩溃。
- **原因**：浏览器在请求未完成时关闭连接，代理尝试写入已关闭的连接。
- **解决**：代理捕获 `ConnectionAbortedError`、`ConnectionResetError`、`BrokenPipeError`。
- **如何避免**：代理服务器的所有写响应操作都要有异常捕获，不能让单个请求异常导致整个服务崩溃。

### #8 bat 脚本中文乱码

- **日期**：早期开发
- **现象**：双击 bat 文件后报错 `'xxx' 不是内部或外部命令`，中文全部变成乱码。
- **原因**：Windows cmd 默认使用 GBK 编码，而文件保存为 UTF-8。
- **解决**：bat 文件中不要使用中文，全部用英文。
- **如何避免**：所有 bat 脚本一律使用英文，参考本文档"开发规则"第 4 条。

### #9 Python `send_response()` 自动添加 Server/Date 头

- **日期**：早期开发
- **现象**：代理响应头中出现重复的 Server 或 Date 头。
- **原因**：Python `http.server` 的 `send_response()` 会自动添加 `Server` 和 `Date` 头。
- **解决**：不要在代理中手动添加 `Server` 和 `Date` 头。
- **如何避免**：使用框架/库的 API 时，先了解其默认行为，避免与手动添加的内容重复。

### #10 端口冲突导致服务启动失败

- **日期**：早期开发
- **现象**：启动 HTTP 服务器时报 `Address already in use`。
- **原因**：之前的进程没有正确退出，端口仍被占用。
- **解决**：启动前用 `netstat -ano | findstr :<端口>` 检查，用 `taskkill /PID <ID> /F` 清理。
- **如何避免**：服务异常退出后，再次启动前先检查端口占用情况。
